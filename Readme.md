Завдання 1. UART + Парсинг GPS

    Опис роботи: 
        перевірка коректності вхідного рядка
        валідація CRC (контрольної суми)
        розбір рядка на поля
        перетворення координат з NMEA формату на десятковий
        обробка помилок

    Замість використання фізичного UART-порту - дані отримуються через стандартний вхідний потік (stdin).
    Таким чином можно імітувати роботу з GPS-модулем без необхідності підключення апаратного забезпечення.

    Приклад вхідної строки:
        $GPGGA,123519,4807.038,N,01131.000,E,1,08,0.9,545.4,M,46.9,M,,*52

    Етапи обробки:
        1. Перевірка формату ($GPGGA)
        
        2. Перевірка CRC
        
        3. Поділ рядка по ","
        
        4. Парсінг:
            широти (latitude)
            довготи (longitude)
            кількості супутників

        5. Конвертація координат

    Запуск: gpsParsing.exe

Завдання 3. ROS2-нода з UART
    Запускати в середовищі Ubuntu 22.04 (WSL) з встановленим ROS2 Humble.
    
    Нода:
        приймає рядок NMEA (stdin)
        валідує (CRC)
        парсит координати
        публікує у /gps_position

    Формат message:
        float32 latitude 
        float32 longitude 
        uint8 satellites

    Приклад вхідної строки:
        $GPGGA,123519,4807.038,N,01131.000,E,1,08,0.9,545.4,M,46.9,M,,*52

    Запуск:
        Термінал 1:
            mkdir -p ~/ros2_ws/src
            cd ~/ros2_ws/src
            git clone <репозиторій>
            
            cd ~/ros2_ws
            
            source /opt/ros/humble/setup.bash
            
            colcon build

            source ~/ros2_ws/install/setup.bash 
            ros2 run gps_package gps_node

        Термінал 2:
            source /opt/ros/humble/setup.bash 
            source ~/ros2_ws/install/setup.bash 
            ros2 topic echo /gps_position
